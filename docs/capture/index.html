Capturing Photos

Press P (keyboard), X / Square (gamepad) or the SNAP button, or call PhotoModeController.CapturePhoto() from script. The UI hides itself for the shot, the frame is rendered offscreen, encoded to PNG and saved to disk, and a thumbnail toast confirms it.

Supersampling¶

The Capture tab offers 1× / 2× / 4× output resolution (PhotoModeController.Supersample, default from PhotoModeSettings.defaultSupersample). A 4× capture of a 1080p view renders at 7680×4320. Temporal anti-aliasing and dynamic resolution are disabled for the duration of every capture, so the output is artifact-free and exactly the requested size.

Heavy captures show a badge

Supersampled captures block the main thread for the render + encode. A static PROCESSING… badge appears automatically for 2×/4× shots. It lives on the overlay canvas and can never end up in the photo. Expect multiple seconds at 4× from 1080p; measured timings are on the Performance page.

The readback uses AsyncGPUReadback where the platform supports it, with a synchronous fallback elsewhere.

Where photos go¶

Setting Result
PhotoModeSettings.customSavePath empty (default) {Application.persistentDataPath}/Photos
customSavePath set that directory, used verbatim

PhotoModeController.SaveDirectory returns the resolved directory at runtime. The save-path row on the Capture tab is clickable and opens the folder; clicking the capture toast reveals the new file (Windows Explorer reveal; folder open on other platforms).

File naming: {ProductName}_{yyyy-MM-dd_HH-mm-ss}.png, with the product name sanitized for the file system. Collisions get a counter suffix: Game_2026-06-11_12-00-00_001.png.

Transparent background¶

Toggle Transparent BG on the Capture tab (or set PhotoModeController.TransparentCapture) to capture with a clean alpha channel, useful for compositing screenshots over other art.

Warning

  • Transparent frames render without post-processing (post effects destroy alpha), so your image adjustments do not appear in transparent shots.
  • Not available on HDRP: the toggle is hidden there and a transparent request is ignored at the backend boundary. URP and both Built-in backends support it. Details on Pipelines.

Hiding things for the shot¶

Toggle Hotkey What it does
Hide UI H / Y (Triangle) Hides the photo mode chrome for a clean preview. Capture feedback (flash, toast) stays visible.
Hide Game UI Capture tab Hides the canvases registered on the controller (GameUiCanvases list + gameUiTag). On by default when autoHideGameUi is set.
Hide Player Capture tab Disables the renderers of the registered player objects (PlayerObjects list + playerTag). Animator/physics state is untouched, so unhiding never pops.
Hide Layers Capture tab Removes PhotoModeSettings.hiddenLayers from the photo camera's culling mask.

All of these are restored exactly on exit. Scripts can drive them through PhotoModeController.Visibility (SetGameUiHidden / SetPlayerHidden / SetLayersHidden).

Reacting to captures¶

using PhotoModePro;
using UnityEngine;

void OnEnable()
{
    PhotoModeController.Instance.OnPhotoCaptured += HandlePhoto;
}

void HandlePhoto(string path, Texture2D thumbnail)
{
    Debug.Log($"Photo saved to {path}");
    // e.g. push the thumbnail into your gallery UI, or upload the file.
}

The same event exists as a UnityEvent (On Photo Captured) on the controller for Inspector wiring. IsCapturing is true while a capture is in flight. ExitPhotoMode() no-ops during that window, and the toggle key is ignored.
