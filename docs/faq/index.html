FAQ

Quick answers to the questions buyers ask most. For symptom-driven help, see Troubleshooting; for version migrations, see the Upgrade Guide.

Getting started¶

How long does it take to add photo mode to my game?¶

About 15 minutes. Drag the generated PhotoModePro.prefab into your scene, add your HUD canvases and player root(s) on the PhotoModeController component, and press your toggle key in Play mode. The Quick Start walks through every step.

What does the Setup Wizard do?¶

Tools > Photo Mode Pro > Setup Wizard runs green/yellow/red checks on your project — render pipeline, effects backend, input handling, Cinemachine, TMP Essential Resources and demo materials — and offers one-click fixes for anything yellow or red. Run it first whenever something looks wrong; it catches the most common setup problems (wrong-pipeline demo materials, missing TMP essentials) before they cost you time.

Do I need to install any other packages?¶

No hard dependencies. Recommended/optional:

  • Input System package — recommended; the legacy Input Manager also works with limitations (see below).
  • Post Processing Stack v2 (com.unity.postprocessing) — only on the Built-in Render Pipeline, for the full adjustment set. Without it a minimal fallback backend is used (exposure, contrast, saturation, vignette only).
  • TMP Essential Resources — the UI uses TextMeshPro; import them once via Window > TextMeshPro > Import TMP Essential Resources (the Setup Wizard checks this and the demo builder imports them automatically).

Can I open photo mode from my own code, e.g. a pause menu?¶

Yes:

using PhotoModePro;

PhotoModeController.Instance.EnterPhotoMode();
PhotoModeController.Instance.ExitPhotoMode();

PhotoModeController.Instance.OnPhotoCaptured += (path, thumbnail) =>
    Debug.Log($"Saved {path}");

All lifecycle events also exist as UnityEvents for Inspector-only wiring. See Events and API.

Does opening photo mode pause my game?¶

By default yes — timeScale = 0 with audio paused, and everything restored exactly on exit. A non-pausing "unscaled" mode is available on the settings asset for games that must keep simulating. Audio pausing is controlled separately via PhotoModeSettings.audioPauseMode (AudioListenerPause or None for mixer-snapshot-driven games).

Compatibility¶

Which Unity versions and render pipelines are supported?¶

Unity 2021.3 LTS or newer, verified on 2021.3, 2022.3 and 6000.3. All three render pipelines: URP (12+), HDRP (12+) and the Built-in Render Pipeline (full set with Post Processing Stack v2, minimal fallback without it). The right effects backend is detected automatically at runtime. See the Support Matrix and Pipelines.

Does it work with both the new Input System and the legacy Input Manager?¶

Yes. The Input System package is the recommended path and is fully supported. The legacy Input Manager works too, with one documented limitation: gamepad right-stick look and triggers are unavailable, because the default InputManager.asset defines no axes for them and the package never edits your ProjectSettings. Keyboard + mouse is fully supported on legacy. Projects with Active Input Handling set to Both are also verified.

Does it work with Cinemachine?¶

Yes. Photo mode spawns its own free camera (a clone of your main camera) on enter and hands control back on exit, so it sits alongside Cinemachine rather than fighting it — your virtual cameras keep owning the gameplay camera. The Setup Wizard includes a Cinemachine check. See Integrations for details.

What about WebGL, mobile and consoles?¶

  • Windows / macOS / Linux standalone — fully supported, Mono and IL2CPP (IL2CPP verified with Managed Stripping Level = High).
  • WebGL — not supported in this version (capture-to-disk); a known limitation and roadmap item.
  • Mobile — untested / roadmap; there is no touch UI yet.
  • Consoles — untested / roadmap; the gamepad input layer is console-style, but no platform claims are made.

Capture¶

Where are photos saved?¶

{Application.persistentDataPath}/Photos by default, as timestamped, collision-safe PNGs. Override the folder via PhotoModeSettings.customSavePath. In-game, clicking the capture toast (or the save-path row on the CAPTURE tab) opens the folder — on Windows with the new photo selected. See Capture.

Why is the Transparent BG toggle missing on HDRP?¶

HDRP cannot produce clean alpha without an alpha-capable color buffer format on the HDRP asset — a project-wide setting the package will not edit, so the capability is honestly reported as unsupported and the toggle is hidden. URP and both Built-in backends support transparent capture; transparent frames render without post-processing so the alpha stays clean. See Pipelines.

How large can captures be?¶

1x, 2x or 4x supersampled — a 4x capture of a 1080p view renders at 7680×4320. Heavy supersampled captures show a "PROCESSING…" badge automatically while the blocking render runs. See Performance for measured wall-times.

Does photo mode allocate GC garbage every frame?¶

No. Photo mode does zero per-frame managed (GC) allocations while open and idle and while adjusting sliders, after the first-open warmup frame that builds caches. While closed, the controller only polls the toggle binding. This is asserted by allocation gates in the test suite. See Performance.

UI & input¶

Can I rebind the keys?¶

Yes — the toggle, hide-UI and capture hotkeys (and pause behavior, camera limits, save folder and more) live on the PhotoModeSettings asset. Duplicate it or create one via Assets > Create > Photo Mode Pro > Settings. When you change a hotkey in the Inspector, an editor dialog offers to update the matching on-screen hint chips for you.

Can I restyle the UI to match my game?¶

Yes — the whole UI restyles from a single PhotoModeUITheme asset: colors, fonts, sprites, motion (hover/press scales, transition durations) and UI audio (hover/click/shutter sounds). See Theming.

Is the UI fully gamepad-navigable?¶

Yes — D-pad navigates, sticks stay reserved for the camera (AAA split scheme). The first control of the active tab is selected the moment photo mode opens, and navigation can never go dead: pressing a navigation input while nothing is selected re-selects the first control automatically.

Demo & assets¶

Why does the demo render pink/magenta?¶

The demo materials were generated for a different render pipeline than the one your project is using. Since 1.4.0 the demo is pipeline-aware: opening the demo scene under a mismatched pipeline offers "Rebuild demo for current pipeline" — accept it, or run Tools > Photo Mode Pro > Build Demo Assets manually. See Troubleshooting.

Can I delete the Demo folder?¶

Yes — it is safe to delete. Everything demo-related (scene, third-party art and audio, demo builders and demo tests) lives under Assets/PhotoModePro/Demo/, and nothing outside it references that folder. One thing to know: the generated controller/UI prefabs, settings, theme and presets live under Demo/Generated/, so if you delete the demo, create your own settings asset (Assets > Create > Photo Mode Pro > Settings) and add the PhotoModeController component to a GameObject directly.

Who made the demo art, and can I ship it?¶

All bundled third-party demo assets are CC0 (public domain) — from Quaternius, Kenney, Poly Haven, ambientCG and OpenGameArt — with full provenance in Demo/ThirdParty/ATTRIBUTIONS.md. You may use them, but remember: the product you bought is the photo mode system; the demo art exists to show it off, and the same CC0 assets are freely available from their original sources.

A Welcome window appeared after import — will it keep popping up?¶

No. It shows once per machine on first import (gated by an EditorPrefs flag) and is never re-shown, including across updates. You can reopen it any time via Tools > Photo Mode Pro > Welcome.

How do I get support?¶

Use Tools > Photo Mode Pro > Report a Bug, or email support.dan.max@gmail.com. Before reporting, run Tools > Photo Mode Pro > Run Self-Test and attach the generated PhotoModeSelfTest.json — it captures the environment details we need. See also Troubleshooting.
