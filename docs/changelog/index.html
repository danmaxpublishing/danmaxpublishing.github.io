Changelog

All notable changes to Photo Mode Pro are documented here. The format follows Keep a Changelog.

[1.4.0] - 2026-06-13¶

Added¶

  • Setup Wizard (Tools → Photo Mode Pro → Setup Wizard): green/yellow/red project checks — render pipeline, effects backend, input handling, Cinemachine, TextMesh Pro essentials, demo materials — with one-click fixes where automatable, and an Add Photo Mode To Scene button that drops the configured prefab into the open scene and selects it.
  • Editor menu root Tools → Photo Mode Pro: Setup Wizard, Add Photo Mode To Scene, Run Self-Test, Build Demo Assets, Open Documentation, Report a Bug (pre-filled email), Welcome and About (version + latest changelog).
  • First-import welcome window, shown once per machine (EditorPrefs-gated with a version-independent key — package updates never re-show it; never in batch mode), linking the wizard, documentation and support.
  • Pipeline-aware demo: demo materials are generated for the ACTIVE render pipeline (Standard / URP Lit / HDRP Lit), so the demo is never pink. Opening the demo scene under a mismatched pipeline offers "Rebuild demo for current pipeline" (persisted don't ask again; never in batch mode); the wizard offers the same fix.
  • Self-test utility (Tools → Photo Mode Pro → Run Self-Test, or -executeMethod PhotoModePro.Editor.PhotoModeSelfTest.Run): verifies compilation, runs the wizard checks headlessly, rebuilds the demo and validates the generated materials, writing a JSON result file.
  • Documentation site at https://supportdanmax.github.io/photo-mode-pro-docs/ (quick start, how-tos, integrations, per-pipeline notes, FAQ, troubleshooting, API reference) plus an offline quick start in the package root.

Fixed¶

  • Aperture slider dead zone (NB-15): the slider was linear in f-stops while depth-of-field blur scales with 1/f-number, so everything visible happened below ~f/3. The slider now spans f/0.7–f/16 on a perceptual log-stop mapping (equal travel = equal stop ratio) and the full travel produces visible change. Out-of-range aperture values from presets or scripts (the old range reached f/32) are clamped when applied — the property still accepts any positive value, no API break.
  • Hotkey hint chips relayout (NB-14): the four stacked rows in the bottom-left corner are replaced by a single centered row of chips along the bottom screen edge (the toolbar moved up to clear it). On canvases too narrow for one row (narrower than ~1.15:1) trailing chips wrap onto a second row stacked above. Chip metrics are unchanged.
  • Focus rings clipped again on wide controls (NB-16/NB-17, regression of NB-04/NB-10): the Phase 3 padding was still present but did not account for the hover/selection scale-up the button feedback applies — on full-width controls the scaled ring crossed the scroll-view mask. Page padding is now computed from the selection-scale invariant, and a geometry regression test proves every ring stays inside the mask at selection scale on the generated prefab.

Fixed (QA pass)¶

  • A pre-existing EventSystem gets its original UI actions back on exit — the D-pad-only navigation scheme photo mode applies no longer persists into the game's own menus.
  • Disabling or destroying the controller mid-capture no longer wedges photo mode open (time scale, cursor, audio, camera and spawned objects are all released; the abandoned capture frees its RenderTexture and textures).
  • Effects backends destroy the volume components they created on exit; long sessions no longer strand a handful of ScriptableObjects per enter/exit cycle.
  • HDRP on Unity 2021.3/2022.1 (HDRP 12/13) compiles again: the photo camera's aperture goes through HDPhysicalCamera there, and the HDRP-14-only focusDistanceMode override is version-guarded (same behavior on all versions).
  • The PPv2 backend's template-layer lookup no longer requires 2021.3.18+.
  • Photo file names are sanitized with a platform-independent character set, so captures named on macOS/Linux stay portable to Windows.
  • The capture thumbnail no longer leaks in UI-less (pure API) integrations, and its ownership is documented on OnPhotoCaptured.

Changed¶

  • PhotoModeAdjustments.ApertureMin/ApertureMax constants changed from 1/32 to 0.7/16 (see the aperture fix above). Existing presets need no action: values outside the range clamp when applied.
  • Additive scene loads no longer auto-close photo mode (streamed chunks are harmless); scene unloads and full scene replacement still do.
  • EnterPhotoMode() on a disabled controller is ignored with a warning instead of opening a dead session (no input, frozen cursor logic).

[1.3.0] - 2026-06-12¶

Added¶

  • Unity 2021.3+ support. The minimum Unity version drops from 6000.0 to 2021.3 LTS; the package is verified on 2021.3, 2022.3 and 6000.3, on URP 12/14/17, Built-in (PPv2 and fallback) and HDRP, with the new Input System, the legacy Input Manager and Active Input Handling = Both.
  • PhotoModeSettings.audioPauseMode (AudioPauseMode.AudioListenerPause / AudioPauseMode.None): with None the package never touches audio, so mixer-snapshot-based games can drive their own transitions from the enter/exit events. Migration note: this replaces the pauseAudio bool; projects that had unchecked pauseAudio must select None once on their settings asset.
  • PhotoModeController.ExitedThisFrame: true only during the frame photo mode closed. Gameplay code sharing bindings with photo mode (e.g. Space as jump and fly-up) skips input that frame so the closing press cannot bleed through; the demo character uses the pattern and QuickStart documents it.
  • Missing-EventSystem handling: if the scene has no EventSystem when the UI opens, photo mode creates one (matching the active input handling), logs a single info message and removes it on exit — the scene is left exactly as found.
  • Scene-change safety: loading or unloading a scene while photo mode is open now auto-closes it cleanly (time scale, audio, cursor, visibility restored first; an in-flight capture is abandoned), with an OnDestroy safety net if the controller itself is torn down.
  • PhotoModeUIController.EnsureSelection(): selects the first control of the active category when nothing is selected (used by the new navigation fallback, available to integrators).
  • IL2CPP build proof: a smoke-test player (IL2CPP, Managed Stripping Level = High) opens photo mode, applies a preset, captures at 2x and verifies the PNG — run as part of the release checks.

Fixed¶

  • Arrow/D-pad navigation is live the moment photo mode opens: the first control of the active tab is auto-selected (focus ring visible), and pressing a navigation key while nothing is selected (e.g. after a mouse click) selects it again — navigation can never go dead.
  • The side panel scroll view now follows keyboard/gamepad selection with minimal auto-scroll: it moves just enough to reveal the selected control plus its focus ring and never moves when the selection is already visible.
  • Focus rings are no longer clipped by the scroll-view mask: pages pad their content (8 px) so rings fit fully inside the viewport on all sides (toggles, preset buttons, Fly/Orbit, 1x/2x/4x, capture-folder row).
  • The NONE preset button is gone: Clean is now the first preset and the active default (the current-preset label reads "Clean" on open and after any reset). ApplyPreset(null) remains the programmatic reset and maps to the same state — no API break.
  • Hotkey hint chips (photo mode bar and demo HUD) scaled ×1.5 (font 11 → 16.5) for readability; the photo-mode bar regroups into four short rows so nothing collides with the toolbar at 21:9, 16:9 or 4:3.
  • Hide Player now disables the player's Renderer components instead of deactivating the GameObject, so CharacterController/Animator/physics state survives hide → unhide → exit with zero pops or jumps (regression-tested for exact position/velocity/grounded equality).
  • The D-pad UI actions asset is built once and reused; repeated open/close no longer accumulates ScriptableObjects.
  • Slider value labels use cached label lookups and TextMeshPro's non-allocating numeric SetText, making slider drags GC-free.

Performance¶

  • Zero per-frame GC allocations while photo mode is open and idle and while adjusting sliders (asserted by allocation gates in the test suite); no per-frame work besides the toggle poll while closed.

[1.2.1] - 2026-06-12¶

Fixed¶

  • All side-panel pages now live in a shared ScrollRect with a slim scrollbar; the mouse wheel scrolls the panel when the pointer is over UI and zooms the camera otherwise.
  • The expanding soft-glow control highlight is replaced by a thin (~2 px) snug outline ring — circular on slider knobs and toggles, rounded-rect on buttons and tabs — that never overlaps neighboring controls or hides the toggle checkmark. Mouse clicks clear the EventSystem selection, so the focus ring appears only during keyboard/gamepad navigation (and sliders now show it).
  • Wheel zoom updates the FOV slider and readout live (two-way sync; roll too).
  • The FOV slider range follows PhotoModeSettings.minFov/maxFov instead of the hardcoded 10–120.
  • The Focus Dist slider uses a logarithmic mapping over 0.1–100 m (half the travel covers the visibly-focus-pulling 0.1–3 m); the readout stays in meters.
  • Focus Dist and Aperture rows hide while Depth of Field is off.
  • HDRP: Focus Dist and Aperture sliders now work — FOV smoothing was silently resetting the physical camera every frame (fieldOfView writes); the camera is now driven via focalLength while physical properties are in use.
  • Capture feedback (white flash, shutter sound, thumbnail toast) is exempt from Hide UI on every pipeline: the root canvas stays enabled and only the chrome (toolbar, panel, hint bar) hides.
  • 2x/4x captures show a static "Processing…" badge for the duration of the blocking render; the badge never appears in the captured photo.
  • Film grain response is normalized across pipelines (URP boosted toward the PPv2 look) and PPv2 vignette response is boosted toward the URP look.
  • The demo character no longer scale-pops when jumping or after Hide Player: the animator now uses de-scaled copies of the imported clips.
  • Sprint (Shift) now multiplies Z/C roll speed by the same sprintMultiplier as movement.

Added¶

  • Clickable capture toast: opens the file browser with the new photo selected (Windows Explorer reveal; folder open elsewhere). The save-path row on the CAPTURE tab is clickable too and opens the capture folder.
  • Hint bar chips for "Space/Ctrl · Up/Down" and "Z/C · Roll" (7 chips total).
  • Scene-floating text (hint labels, toolbar Q/E hints, toast filename) gets a thin black TMP outline + underlay for readability over bright surfaces.
  • The demo HUD is rebuilt from the photo-mode UI building blocks (TMP text, rounded keycap chips, theme colors).
  • Changing a hotkey on the settings asset offers to update the matching UI prefab hint chips (editor dialog).

[1.2.0] - 2026-06-12¶

Added¶

  • Built-in Render Pipeline support: a Post Processing Stack v2 backend (own PostProcessVolume + profile + layer on the photo camera, full adjustment set) when com.unity.postprocessing is installed, and a minimal fallback backend (exposure / contrast / saturation / vignette via one internal full-screen shader) when it is not.
  • HDRP support: an HDRP volume backend (ColorAdjustments, WhiteBalance, Vignette, FilmGrain, DepthOfField) using the physical camera for depth of field (aperture drives Camera.aperture). Exposure maps to ColorAdjustments.postExposure — relative EV, same semantics as URP.
  • Capability system: EffectCapabilities flags (PhotoModePro.Effects), reported per backend via the new IPhotoModeEffectsBackend.Capabilities property and surfaced through PhotoModeController.ActiveCapabilities (live prediction while photo mode is inactive). The runtime UI hides controls — and whole categories — for unsupported effects; gamepad navigation never lands on a dead control.
  • Backend selection: automatic detection from the active render pipeline asset, with a manual PhotoModeSettings.backendOverride (Auto/Urp/Hdrp/PostProcessingV2/BuiltInFallback). An unavailable forced backend logs one warning and falls back to Auto.
  • package.json (com.danmax.photomodepro) so the asset folder can be consumed as a local UPM package; PipelineTestSceneBuilder editor bootstrap that builds a minimal pipeline-appropriate test scene in any host project.
  • Capture path audit: temporal anti-aliasing (URP and HDRP) is disabled for the duration of every capture and dynamic resolution is forced off, so 1x/2x/4x supersampled output is artifact-free and exactly the requested size.

Changed¶

  • Breaking for custom backend implementors: IPhotoModeEffectsBackend gains EffectCapabilities Capabilities { get; } and Initialize() is now Initialize(Camera photoCamera). Backends that ignore the camera can simply discard the argument.
  • Transparent capture is clamped at the backend boundary: the request is ignored when the active backend cannot produce clean alpha (HDRP). On both Built-in variants transparent capture disables the system's PPv2 volume/layer or skips the fallback blit for the shot, matching URP semantics.

Known limitations¶

  • HDRP reports TransparentCapture as unsupported (clean alpha would require an alpha-capable color buffer format on the HDRP asset, which the package does not edit).
  • The Built-in fallback backend (no PPv2) supports exposure, contrast, saturation and vignette only; temperature, film grain and depth of field controls are hidden.

[1.1.0] - 2026-06-11¶

Added¶

  • Demo upgrade: the primitive corridor is replaced by a stylized forest-clearing showcase scene (CC0 third-party assets, provenance in Demo/ThirdParty/ATTRIBUTIONS.md), still built 100% programmatically by DemoSceneBuilder.
  • Rigged, animated low-poly adventurer with a from-scratch third-person controller (ThirdPersonCharacter): camera-relative WASD / left-stick movement, walk/run sprint, smooth acceleration, gravity and jump; freezes cleanly while photo mode is active.
  • Third-person follow camera (ThirdPersonFollowCamera) with mouse / right-stick orbit, clamped pitch, smoothed follow and sphere-cast collision; hands off to and restores from the photo mode free camera.
  • 2K golden-hour HDRI skybox with matched directional sun, URP fog and atmosphere tuned for the DoF / filter presets.
  • Campfire hero prop: scripted flickering point light (CampfireFlicker), code-built fire and smoke particles, drifting fireflies and dust motes.
  • Ambient forest audio plus positional campfire crackle — audibly paused by photo mode's AudioListener.pause showcase.
  • Wandering ambient critters on scripted waypoint patrols (CritterPatrol).
  • Physics props (crates/barrels) the character can bump around.
  • Deterministic seeded scatter for trees/rocks/grass (DemoScatterMath), code-generated forest-floor mesh and code-built animator controllers.

Removed¶

  • SimplePlayerMovement (superseded by ThirdPersonCharacter).

[1.0.0] - 2026-06-11¶

Added¶

  • Photo mode controller with EnterPhotoMode() / ExitPhotoMode() public API, static PhotoModeController.Instance accessor, C# events and UnityEvent mirrors (OnEntered, OnExited, OnPhotoCaptured(path, thumbnail)).
  • Configurable pause behavior: timeScale = 0 with audio pause (default) or non-pausing "unscaled" mode.
  • Free photo camera with fly and orbit modes, sphere clamping around the activation point (15 m default), optional sphere-cast collision, FOV control, roll, sprint and exponential smoothing.
  • Input abstraction supporting both the new Input System and the legacy Input Manager behind version defines; AAA split control scheme (sticks drive the camera, D-pad navigates the UI; RMB-hold mouse look).
  • Image adjustments in natural units via a code-built URP Volume: exposure (EV), contrast, saturation, color temperature, vignette, film grain and bokeh depth of field (focus distance + aperture), decoupled from URP behind IPhotoModeEffectsBackend.
  • Six built-in filter presets (Noir, Vivid, Warm, Cool, Vintage, Clean) plus a "None" reset entry and automatic "Custom" state.
  • Visibility toggles for game UI canvases, player objects and camera layers, with exact state restoration on exit.
  • Supersampled capture at 1x/2x/4x with async GPU readback, PNG output to {persistentDataPath}/Photos (configurable), timestamped collision-safe file names, optional transparent-background capture and thumbnail generation.
  • Reskinnable UGUI panel (Camera / Effects / Presets / Capture tabs) themed by a PhotoModeUITheme asset; fully gamepad-navigable; hide-UI hotkey; capture flash and thumbnail toast feedback.
  • DemoSceneBuilder editor tool that programmatically generates the demo scene, UI prefab, controller prefab, settings, theme and preset assets.
  • EditMode and PlayMode test suites (Unity Test Framework).
