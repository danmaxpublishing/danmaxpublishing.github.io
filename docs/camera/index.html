Free Camera

When photo mode opens, the main camera is cloned into a free photo camera driven by a FreeCameraController. It runs on unscaled time (so it works while the game is frozen) and allocates nothing per frame.

Fly vs orbit¶

Mode Behavior
Fly (default) Free flight: WASD / left stick moves, Space / Left Ctrl (triggers) elevate, mouse / right stick looks.
Orbit The camera circles the activation point (where photo mode was opened) at an adjustable distance. Look input orbits; forward/back input changes the orbit distance.

Switch in the UI's Camera tab (Fly / Orbit buttons) or from script. Switching preserves the current framing. The starting mode comes from PhotoModeSettings.defaultCameraMode.

Controls¶

Action Keyboard / Mouse Gamepad
Move / fly WASD, Space up, Left Ctrl down Left stick, triggers up/down
Look Hold right mouse button Right stick
Sprint Left Shift Left stick click
Roll Z / C Roll slider (Camera tab)
Zoom (FOV) Mouse wheel FOV slider (Camera tab)

Notes:

  • Sprint multiplies both move speed and roll speed by PhotoModeSettings.sprintMultiplier.
  • Wheel zoom only acts when the pointer is not over the UI (over the side panel the wheel scrolls the panel). It moves the FOV slider live, two-way.
  • The FOV slider range follows PhotoModeSettings.minFov / maxFov (10–120° by default).

Limits: sphere clamp and collision¶

Both are configured on the settings asset:

Setting Meaning
maxDistance Max distance from the activation point, in meters (sphere clamp). Default 15; 0 = unlimited.
enableCollision Sphere-cast the camera against geometry so it cannot clip through walls. Off by default.
collisionRadius Radius of that sphere cast (default 0.2 m).
collisionMask Layers the camera collides with.

Note

The sphere clamp is your spoiler/exploit guard. It keeps players from flying the camera across the level. The activation point is exposed as FreeCameraController.Pivot.

Driving the camera from script¶

PhotoModeController.PhotoCamera returns the FreeCameraController while photo mode is active, and null otherwise:

using PhotoModePro;
using UnityEngine;

FreeCameraController cam = PhotoModeController.Instance.PhotoCamera;
if (cam != null)
{
    cam.Mode = PhotoModeCameraMode.Orbit;  // or call cam.SetMode(...)
    cam.Fov = 35f;                          // clamped to settings.minFov..maxFov
    cam.Roll = 10f;                         // degrees, wrapped to -180..180
    cam.MoveSpeed = 12f;                    // meters/second, min 0.1

    Debug.Log($"Orbit pivot: {cam.Pivot}");
}

The component lives on the spawned photo camera GameObject, so cam.transform / cam.GetComponent<Camera>() give you the actual camera if you need to read its pose or projection.

Physical cameras (HDRP)

While the camera uses physical properties, the controller drives zoom via focalLength instead of fieldOfView. Writing fieldOfView would silently switch the camera back to non-physical mode and break HDRP's depth of field. Use the Fov property and let the controller pick the right path.

Position and rotation are smoothed exponentially (PhotoModeSettings.smoothing, 0 disables smoothing), so script writes to Fov/Roll ease in rather than snapping.
