About

Hi, I'm Dan.
I build the tools I always wished someone else had.

DanMaxPublishing is one senior Unity developer with more than a decade of commercial experience. Over the years I've shipped everything from XR applications with tens of thousands of downloads and mobile multiplayer titles built on Photon Fusion and PlayFab, to AI-driven interactive characters and a top-rated golf simulation — and along the way I've led teams as CTO and Unity Lead, built custom editor tooling that halved iteration times, and learned exactly what separates an asset you fight with from an asset you forget is there.

That's the standard every DanMaxPublishing release is held to: clean architecture, thorough documentation, careful performance profiling, and an API you can extend without reading the source. If a tool isn't something I'd confidently drop into my own production project, it doesn't ship.

And here's my promise on support: if you hit a problem, you'll hear back from me — the person who actually wrote the code. I've felt the pain of half-finished assets, missing docs and support tickets that vanish into the void. That's not how this store runs.

House rules

Every asset follows the same rules

No garbage where it doesn't belong

Per-frame allocations are bugs, and the test suite treats them that way. Performance claims are gated by tests, not adjectives.

No undocumented behavior

A full hosted manual, XML docs on every public member, and store pages that tell you exactly what you're getting — including what a pipeline can't do.

No assumptions about your project

Both input systems, all three render pipelines, with or without an EventSystem, Cinemachine-friendly. Full source, asmdefs, demo scenes that actually demonstrate.
