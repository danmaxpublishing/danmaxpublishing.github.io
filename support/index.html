Step 2 · Built-in diagnostics

Let the asset diagnose itself

Run Self-Test

Tools → Photo Mode Pro → Run Self-Test checks your project (pipeline, packages, input handling, prefab wiring) and writes a PhotoModeSelfTest.json report. Attach it to any support request and you've answered the first five questions before they're asked.

Report a Bug

Tools → Photo Mode Pro → Report a Bug opens a pre-filled email with the product and version already in the subject line. Add your repro steps and the self-test report, hit send.

Step 3 · Talk to a human

Contact channels

Email

support.dan.max@gmail.com

The primary channel, for anything: bugs, integration questions, feature requests, invoice issues. Replies come from the developer, usually within a business day or two.

GitHub Issues

Open the issue tracker

Prefer bugs in the open? File an issue so other users can find it, follow it, and see the fix version when it ships.

What a great bug report contains

  • Unity version and render pipeline (URP / HDRP / Built-in ± PPv2)
  • Photo Mode Pro version (Tools → Photo Mode Pro → About)
  • The PhotoModeSelfTest.json report
  • Steps to reproduce, ideally starting from the demo scene
  • What you expected, and what happened instead (screenshots welcome)
