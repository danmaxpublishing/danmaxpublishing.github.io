Quick Start

Two ways in: the wizard path (about 2 minutes) or the manual path (about 10).

Requirements¶

  • Unity 2021.3 LTS or newer (verified on 2021.3, 2022.3 and 6000.3).
  • Any render pipeline: URP (12+), HDRP (12+) or Built-in. On Built-in, install com.unity.postprocessing (Post Processing Stack v2) for the full adjustment set; without it a minimal fallback backend is used. The right backend is detected automatically at runtime (override via PhotoModeSettings.backendOverride).
  • Optional: the Input System package (recommended). The legacy Input Manager also works; see Integrations for its gamepad limitations.

The wizard path (2 minutes)¶

  1. Import Photo Mode Pro. A welcome window appears on first import (once per machine), or open Tools → Photo Mode Pro → Setup Wizard any time.
  2. The wizard checks your render pipeline, effects backend, input handling, optional packages, TextMesh Pro essentials and the demo materials, each with a green / yellow / red light, a one-line explanation and a Fix button where a fix can be automated (importing TMP essentials, rebuilding the demo for your pipeline).
  3. Open your scene and press Add Photo Mode To Scene. The configured prefab (settings, six presets, UI) lands in the scene, selected.
  4. Press Play, then press F (or the gamepad View/Share button). You're in.

No EventSystem? No problem.

If your scene has no EventSystem when photo mode opens, one is created for the duration (matching your active input handling) and removed on exit. The scene is left exactly as found.

The manual path (10 minutes)¶

  1. Drag Assets/PhotoModePro/Demo/Generated/PhotoModePro.prefab into your scene (menu equivalent: Tools → Photo Mode Pro → Add Photo Mode To Scene).
  2. The prefab already references the default PhotoModeSettings, the UI prefab and the six built-in presets. Done. Press your toggle key in Play mode.
  3. Recommended wiring on the PhotoModeController component:
    • Game Ui Canvases: add your HUD canvases so photo mode can hide them.
    • Player Objects: add your player root(s) for the Hide Player toggle.
  4. To change keys, pause behavior, camera limits or the save folder, duplicate PhotoModeSettings.asset (or create one via Assets → Create → Photo Mode Pro → Settings) and assign it on the controller.

Try the demo¶

  1. Open Assets/PhotoModePro/Demo/PhotoModeDemo.unity: a forest clearing at golden hour with a campfire, wandering animals and a playable adventurer (all bundled third-party assets are CC0; see Demo/ThirdParty/ATTRIBUTIONS.md).
  2. Press Play. Move with WASD (or left stick), orbit with the mouse (or right stick), Shift to run, Space to jump.
  3. Press F to open photo mode. Fire, animals and audio freeze instantly. Try the Warm preset against the sunset, or pull focus with depth of field.

Demo on HDRP or Built-in

The demo generates its materials for your active pipeline. If you open the demo scene under a different pipeline than it was last built for, a dialog offers Rebuild demo for current pipeline (also available from the Setup Wizard). The demo is never pink.

The Demo folder is safe to delete. Everything demo-related lives under Assets/PhotoModePro/Demo/ and nothing outside it references that folder. The generated controller/UI prefabs, settings, theme and presets live under Demo/Generated/; if you delete the demo, create your own settings asset and add the PhotoModeController component directly (the Add Photo Mode To Scene menu does this for you and explains what to assign).

Default controls¶

Action Keyboard / Mouse Gamepad
Toggle photo mode F View / Share
Move / fly WASD, Space up, Left Ctrl down Left stick, triggers up/down
Look Hold right mouse button Right stick
Sprint Left Shift Left stick click
Switch UI category Q / E LB / RB
Roll Z / C Roll slider (Camera category)
Zoom (FOV) Mouse wheel FOV slider (Camera category)
Navigate UI Mouse / arrow keys D-pad, A to press
Hide UI H Y / Triangle
Capture P X / Square

Hotkeys are configurable on the settings asset; editing one offers to update the on-screen hint chips to match.
