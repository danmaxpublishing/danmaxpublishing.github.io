Per-Pipeline Notes

Photo Mode Pro detects the active render pipeline at runtime and picks the right effects backend automatically. Controls for effects the active backend cannot apply hide themselves (PhotoModeController.ActiveCapabilities exposes the same flags to scripts).

Effect availability¶

Effect URP HDRP Built-in (PPv2) Built-in (fallback)
Exposure ✓ ✓ ✓ ✓
Contrast ✓ ✓ ✓ ✓
Saturation ✓ ✓ ✓ ✓
Temperature ✓ ✓ ✓ —
Vignette ✓ ✓ ✓ ✓
Film grain ✓ ✓ ✓ —
Depth of field ✓ ✓ ✓ —
Transparent capture ✓ — ✓ ✓

URP¶

  • Effects run through a code-built global Volume at very high priority — no scene assets are touched, and everything is removed on exit.
  • URP's film grain reads weaker than PPv2's at the same intensity, so the URP backend uses a coarse grain lookup (Large01) with a low luminance response to match the PPv2 look.
  • URP's own depth-of-field aperture parameter floors at f/1.0, so the bottom of the shared f/0.7–f/16 range renders slightly deeper focus on URP than on HDRP (where f/0.7 is the physical camera's true floor).
  • Camera stacking is safe. The photo camera renders as a normal Base camera; your main camera's overlay stack is left untouched while photo mode is open and is intact after exit (asserted by HostileEnvironmentTests.UrpOverlayCameraStack_UnaffectedAfterExit in the shipped test suite).

HDRP¶

  • Depth of field drives the physical camera (Camera.aperture, focalLength); the package keeps physical properties enabled while applying adjustments so FOV smoothing can't silently kill DoF.
  • Exposure maps to ColorAdjustments.postExposure (relative EV, same semantics as URP) rather than HDRP's absolute scene Exposure override, so the slider behaves identically across pipelines and never fights auto-exposure.
  • Transparent capture is unsupported. Clean alpha would require an alpha-capable color buffer format on the HDRP asset, which the package does not edit. The Transparent BG toggle hides automatically.
  • The demo's panoramic skybox material has no effect on HDRP (HDRP uses volume skies); after a rebuild for HDRP the demo uses HDRP's default sky.

Built-in with Post Processing Stack v2¶

  • The backend creates its own PostProcessVolume, profile and PostProcessLayer on the photo camera, giving the full adjustment set.
  • PPv2's vignette reads weaker than URP's at the same intensity, so the slider value is boosted internally toward the URP look; expect subtle character differences between pipelines at identical slider values.

Built-in fallback (no PPv2)¶

  • A single internal full-screen shader provides exposure, contrast, saturation and vignette. Temperature, film grain and depth of field are unavailable and their controls hide automatically.
  • Install com.unity.postprocessing for the full set — the Setup Wizard's effects-backend check links you straight to the Package Manager.

Aperture clamping (all pipelines)¶

The aperture property accepts any positive value, but backends clamp it into f/0.7–f/16 when applying (why — see the effects how-to). Out-of-range values from old presets or scripts are clamped, never errors.

Demo materials¶

Demo materials are generated for the active pipeline (Standard, URP Lit or HDRP Lit). Opening the demo scene under a mismatched pipeline offers Rebuild demo for current pipeline; the Setup Wizard's demo check offers the same rebuild. See Troubleshooting if the demo ever renders pink.
