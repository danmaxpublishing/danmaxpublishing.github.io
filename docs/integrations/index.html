Integrations

Photo Mode Pro is built to drop into an existing game without taking it over. It requires no packages beyond Unity itself and does not edit your ProjectSettings; state is restored exactly on exit. This page covers the common integration points.

Cinemachine¶

Photo Mode Pro has no hard dependency on Cinemachine (the package compiles and runs without it) but plays cleanly with it when present (the Setup Wizard detects and reports Cinemachine in its checks).

What happens on enter when a CinemachineBrain drives your main camera:

  • The main camera (carrying the brain) is disabled, which suspends Cinemachine's hold on the rendered view. The brain component itself is never touched, removed or disabled.
  • The photo camera is a clean clone: it carries no CinemachineBrain, so virtual cameras cannot pull it around. The free camera owns its transform entirely while photo mode is open.

On exit, the main camera's enabled state is restored exactly and Cinemachine resumes control as if nothing happened. Your virtual cameras keep running throughout (handy with the Unscaled pause mode, where blends continue).

Note

If your vcams animate on scaled time, the default TimeScaleZero pause freezes them along with the rest of the game, which is usually exactly what a photo mode should do.

If you use URP camera stacking alongside (or instead of) Cinemachine, your overlay stack is untouched too; see the per-pipeline notes.

Input systems¶

The input layer is abstracted behind version defines and works with all three Active Input Handling settings (the Setup Wizard verifies your configuration):

Project setting Status
Input System Package (new) Recommended. Full keyboard, mouse and gamepad support.
Input Manager (legacy) Supported, with the gamepad limitation below.
Both Supported; the new Input System backend is used.

Legacy Input Manager gamepad limitation

On the legacy Input Manager, gamepad right-stick look and triggers are unavailable: the default InputManager.asset defines no axes for them, and the package does not edit your ProjectSettings. Keyboard + mouse is fully supported; gamepad UI navigation then uses the default Horizontal/Vertical axes, which also drive camera movement. See the support matrix.

Key and gamepad bindings live on the settings asset (toggleKey/toggleButton, captureKey/captureButton, hideUiKey/hideUiButton).

Audio¶

PhotoModeSettings.audioPauseMode picks the strategy:

Mode Behavior
AudioListenerPause (default) AudioListener.pause is set on enter; the prior value is restored exactly on exit, so a game that was already paused stays paused.
None Photo mode never touches audio. Drive your own transitions from the enter/exit events.

The mixer-snapshot pattern with None:

using PhotoModePro;
using UnityEngine.Audio;

[SerializeField] AudioMixerSnapshot pausedSnapshot;
[SerializeField] AudioMixerSnapshot gameplaySnapshot;

void Start()
{
    var controller = PhotoModeController.Instance;
    controller.OnEntered += () => pausedSnapshot.TransitionTo(0.1f);
    controller.OnExited  += () => gameplaySnapshot.TransitionTo(0.1f);
}

UI sounds (hover, click, shutter) are separate and can be routed into a mixer group of your choice via the theme; see Theming.

Pause menus and game-state machines¶

Treat photo mode as one more state your game can be in, owned by your state machine:

using PhotoModePro;

public class GameStateMachine : MonoBehaviour
{
    void Start()
    {
        var photoMode = PhotoModeController.Instance;
        photoMode.OnEntered += () => SetState(GameState.PhotoMode);
        photoMode.OnExited  += () => SetState(GameState.Paused); // back to the menu
    }

    public void OnPhotoModeButtonClicked()
    {
        HidePauseMenu();
        PhotoModeController.Instance.EnterPhotoMode();
    }

    // ...
}

Practical notes:

  • Already-paused games: photo mode records Time.timeScale on enter and restores that exact value on exit. Entering from a timeScale = 0 pause menu and exiting back to it is fully supported.
  • Disable the built-in toggle if you want exclusive control: the controller polls its toggle binding whenever a settings asset is assigned. Pick a toggle key that does not collide with your menus, or gate entry entirely through your own UI and pick an unused binding.
  • Shared bindings: if any photo mode binding doubles as a gameplay action, use the ExitedThisFrame guard; see Events & API.
  • Hide your HUD: add your HUD canvases to the controller's Game Ui Canvases list (or tag them and set gameUiTag) so photo mode can hide them; they are restored exactly on exit.
  • Scene changes: replacing or unloading a scene while photo mode is open auto-closes it cleanly, so loading screens and level transitions need no special handling. Additive loads (streamed chunks) don't interrupt the session.
