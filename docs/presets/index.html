Presets

A preset is a PhotoModePreset asset: a display name plus one full set of adjustment values. There are no LUT textures or shaders involved, just plain data, so presets stay tiny and diff cleanly in version control.

The built-in six¶

Preset Look
Clean Neutral: every value at its default. First in the list and the active default.
Noir High-contrast desaturated black & white.
Vivid Punchy contrast and saturation.
Warm Golden-hour temperature shift.
Cool Blue-shifted, moody.
Vintage Faded grain-and-vignette film look.

Clean is the default state: PhotoModeController.CurrentPresetLabel reads "Clean" when photo mode opens and after any reset.

Creating your own¶

  1. Assets > Create > Photo Mode Pro > Preset.
  2. Set displayName, the label shown on the Presets tab button.
  3. Fill in the adjustments block (same fields and units as the Effects tab: EV, −100..+100, 0..1, meters, f-stops).

Author presets live

Enter Play mode, dial in a look with the sliders, then copy the values into your preset asset. The component Inspector and the asset use identical fields, so it is a one-to-one transfer.

Aperture range

Aperture values outside f/0.7–f/16 are accepted by the asset but clamped by the effect backends when applied; see the aperture slider.

Offering presets in the UI¶

The controller's Presets list (Inspector field, or the Presets property from script) decides which presets the Presets tab shows, in order:

using PhotoModePro;

var controller = PhotoModeController.Instance;
controller.Presets.Add(myPreset);   // List<PhotoModePreset>

The generated prefab ships with the built-in six already assigned; change the list however you like.

Applying from script¶

using PhotoModePro;

var controller = PhotoModeController.Instance;

controller.ApplyPreset(myPreset);   // copies values in, re-applies effects
controller.ApplyPreset(null);       // programmatic reset -> neutral "Clean" state

ApplyPreset copies the preset's values into controller.Adjustments, pushes them to the effects backend and raises OnAdjustmentsReplaced (so the UI sliders re-seed). ApplyPreset(null) resets every value to neutral, identical to the Clean preset.

The label: Clean, a preset name, or "Custom"¶

CurrentPresetLabel tracks where the current look came from:

State Label
Neutral (open, or after ApplyPreset(null)) "Clean"
A preset was applied the preset's displayName
Any slider moved (or NotifyAdjustmentsEdited() called) after that "Custom"

The label is sticky for the play session, like the adjustment values themselves. The UI shows it so players always know whether they are on a stock look or have drifted from it.
